# Folder: WheelPicker

> Trigger: When this folder's structure/responsibilities/file list changes, update this document.
> 触发条件:当本文件夹的结构/职责/文件列表变化时,更新此文档。

This folder contains the app scene, the immersive demo screen, and the package-consumer shell. / 本目录包含应用场景、沉浸式示例界面，以及包消费者外壳。  
It consumes `WheelPickerKit` from the local Swift package and presents the picker as an edge-to-edge warm arc composition using the package's explicit initial-selection API. / 它消费本地 Swift Package 中的 `WheelPickerKit`，并使用包公开的初始值 API 将选择器展示为贴边的暖色圆弧构图。  
The invariant is that app-facing composition stays here, while all shipped picker behavior and default-value repair live under `Sources/`. / 最重要的不变量是应用侧界面构图必须留在这里，而所有真正分发的选择器行为与默认值修正逻辑都必须位于 `Sources/`。

## Files
- `ContentView.swift`: screen - Full-screen edge-to-edge immersive arc demo composition, explicit initial value wiring, and background treatment / 全屏贴边沉浸式圆弧示例的组合、显式初始值接线与背景处理
- `WheelPickerApp.swift`: app root - Application entry scene / 应用入口场景
- `WheelPicker.entitlements`: config - App sandbox entitlements / 应用沙盒权限配置
- `Assets.xcassets/`: child folder - App icons and color assets / 应用图标与颜色资源
- `.folder.md`: local map - Folder responsibilities and file list / 目录职责与文件列表
