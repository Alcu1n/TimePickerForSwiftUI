# Folder: WheelPicker

> Trigger: When this folder's structure/responsibilities/file list changes, update this document.
> 触发条件:当本文件夹的结构/职责/文件列表变化时,更新此文档。

This folder contains the app scene, the immersive demo screen, and the package-consumer shell. / 本目录包含应用场景、沉浸式示例界面，以及包消费者外壳。  
It consumes `WheelPickerKit` from the local Swift package and owns the warm blurred background presentation. / 它消费本地 Swift Package 中的 `WheelPickerKit`，并负责暖色模糊背景的展示外壳。  
The invariant is that app-facing composition stays here, while all shipped picker behavior lives under `Sources/`. / 最重要的不变量是应用侧界面构图必须留在这里，而所有真正分发的选择器行为都必须位于 `Sources/`。

## Files
- `ContentView.swift`: screen - Full-screen immersive arc demo composition and background treatment / 全屏沉浸式圆弧示例的组合与背景处理
- `WheelPickerApp.swift`: app root - Application entry scene / 应用入口场景
- `WheelPicker.entitlements`: config - App sandbox entitlements / 应用沙盒权限配置
- `Assets.xcassets/`: child folder - App icons and color assets / 应用图标与颜色资源
- `.folder.md`: local map - Folder responsibilities and file list / 目录职责与文件列表
