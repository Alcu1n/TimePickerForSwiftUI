# Folder: WheelPicker

> Trigger: When this folder's structure/responsibilities/file list changes, update this document.
> 触发条件:当本文件夹的结构/职责/文件列表变化时,更新此文档。

This folder contains the app scene, the demo screen, and the package-consumer shell. / 本目录包含应用场景、示例界面，以及包消费者外壳。  
It consumes `WheelPickerKit` from the local Swift package and exposes the runnable demo plus tuning UI. / 它消费本地 Swift Package 中的 `WheelPickerKit`，并向应用入口暴露可运行示例与调节界面。  
The invariant is that app-facing layout chrome and demo state live here, while all shipped picker code lives under `Sources/`. / 最重要的不变量是应用侧布局装饰与示例状态必须留在这里，而所有真正分发的 picker 代码都必须位于 `Sources/`。

## Files
- `ContentView.swift`: screen - Timer picker demo composition, scaling, and tuning UI / 计时器选择器示例的组合、缩放与调节界面
- `WheelPickerApp.swift`: app root - Application entry scene / 应用入口场景
- `WheelPicker.entitlements`: config - App sandbox entitlements / 应用沙盒权限配置
- `Assets.xcassets/`: child folder - App icons and color assets / 应用图标与颜色资源
- `.folder.md`: local map - Folder responsibilities and file list / 目录职责与文件列表
